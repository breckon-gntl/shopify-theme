## 2. Collection-Split Tile Upgrade — ATC + Subscribe Toggle + Star Ratings

**Files created/modified:**
- `snippets/collection-split-tile.liquid` — created
- `sections/collection-split.liquid` — modified

### Why a new snippet
`product-grid-item.liquid` (473 lines, shared by collection pages, search, quick-views) intentionally disables instant-add buttons when a product has Recharge selling plans — it can't know if the user wants one-time or subscribe. Editing it would risk breaking swatches and ATC everywhere else. Purpose-built tile is the correct scope.

### collection-split-tile.liquid structure

```liquid
{%- assign variant = product.selected_or_first_available_variant -%}
{%- assign plan_id = product.selling_plan_groups.first.selling_plans.first.id -%}
{%- assign on_sale = false -%}
{%- if variant.compare_at_price > variant.price -%}{%- assign on_sale = true -%}{%- endif -%}

<div class="cs-tile">
  <a href="{{ product.url }}" class="cs-tile__img-wrap">
    <img src="{{ product.featured_media | image_url: width: 600 }}" ...>
  </a>
  <div class="cs-tile__body">
    <a href="{{ product.url }}" class="cs-tile__title">{{ product.title | escape }}</a>

    {%- if product.metafields.reviews.rating.value != blank -%}
      {%- render 'product-rating', product: product, show_rating_count: true -%}
    {%- endif -%}

    <p class="cs-tile__price">
      <span class="price">{{ variant.price | money }}</span>
      {%- if on_sale -%}<span class="compare-at">{{ variant.compare_at_price | money }}</span>{%- endif -%}
    </p>

    {%- if plan_id -%}
      <div class="cs-tile__options">
        <label class="cs-tile__opt">
          <input type="radio" name="cs-purchase-type-{{ product.id }}" value="one-time" checked
            onchange="document.getElementById('cs-sp-{{ product.id }}').disabled=true">
          One-time
        </label>
        <label class="cs-tile__opt">
          <input type="radio" name="cs-purchase-type-{{ product.id }}" value="subscribe"
            onchange="document.getElementById('cs-sp-{{ product.id }}').disabled=false">
          Subscribe &amp; save 10%
        </label>
      </div>
    {%- endif -%}

    <div x-data="productAddButtonForm()">
      {%- form 'product', product -%}
        <input type="hidden" name="id" value="{{ variant.id }}">
        <input type="hidden" name="quantity" value="1">
        {%- if plan_id -%}
          <input type="hidden" name="selling_plan" value="{{ plan_id }}"
            id="cs-sp-{{ product.id }}" disabled>
        {%- endif -%}
        <button type="submit" name="add" class="btn btn--primary cs-tile__btn">Add to cart</button>
      {%- endform -%}
    </div>
  </div>
</div>
```

### Subscription toggle mechanism
The `selling_plan` hidden input starts `disabled`. The browser's `FormData` spec excludes disabled inputs from POST. So:
- **One-time** → input is disabled → not in POST → standard purchase
- **Subscribe** → `onchange` sets `disabled=false` → included in POST → Recharge creates subscription

Each product uses `product.id` in the input ID to avoid DOM collisions across tiles.

## 3. Cart Drawer Auto-Open on Every ATC

**Files modified:**
- `assets/theme.js`
- `sections/sticky-atc.liquid`

### Problem
All ATC buttons showed a mini popdown notification after success. Cart drawer never opened automatically.

### theme.js — productAddButtonForm.handleSuccess()

The decision point is at the bottom of `handleSuccess()`:

**Before (original):**
```javascript
if (window.theme.state.cartOpen || this.isInCartPage || this.isOnCartPage) {
    this.reloadCart();
} else {
    this.openPopdown(variant);
}
```

**After (our change):**
```javascript
const _cartUpdatePromise = this.updateHeaderTotal();  // captured from line above
// ...
const _cartDrawer = document.querySelector('[data-drawer="drawer-cart"]');
if (window.theme.state.cartOpen || this.isInCartPage || this.isOnCartPage) {
    this.reloadCart();
} else if (_cartDrawer) {
    _cartUpdatePromise.then(() => {
        _cartDrawer.dispatchEvent(new CustomEvent('theme:drawer:open', { bubbles: false }));
    });
} else {
    this.openPopdown(variant);  // fallback if no cart drawer exists
}
```

### sticky-atc.liquid — AJAX intercept

The sticky ATC form had no JS submit handler — it caused a full-page redirect to `/cart`. Added inside the existing IIFE before the closing `})()`:

```javascript
var form = document.getElementById('sticky-atc-form');
if (form) {
  form.addEventListener('submit', function (e) {
    e.preventDefault();
    var btn = document.getElementById('sticky-atc-btn');
    if (btn) btn.disabled = true;

    var data = new URLSearchParams(new FormData(form)).toString();

    fetch(window.Shopify.routes.root + 'cart/add.js', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/x-www-form-urlencoded',
        'X-Requested-With': 'XMLHttpRequest'
      },
      body: data
    })
    .then(function (res) { return res.json(); })
    .then(function () {
      document.dispatchEvent(new CustomEvent('theme:cart:change', { bubbles: true }));
      var drawer = document.querySelector('[data-drawer="drawer-cart"]');
      if (drawer) {
        drawer.dispatchEvent(new CustomEvent('theme:drawer:open', { bubbles: false }));
      }
      if (btn) btn.disabled = false;
    })
    .catch(function () {
      if (btn) btn.disabled = false;
    });
  });
}
```

Selling plan is already synced into `#sticky-atc-selling-plan` by the existing sync logic — `FormData` picks it up automatically.

## 4. Bug Fix — Cart Drawer Showed "Empty" After First ATC

**File:** `assets/theme.js`

### Root cause
Race condition. The theme's `CartItems` class keeps an in-memory `this.cart` object. `updateHeaderTotal()` fetches `/cart.js` and dispatches `theme:cart:change` to update `this.cart` — but it's async. We were dispatching `theme:drawer:open` synchronously right after, so the drawer opened while `this.cart` still showed 0 items. `loadHTML()` then read `this.cart.item_count === 0` and called `showEmpty()`.

### Event flow before fix
1. `/cart/add.js` — item added server-side ✓
2. `updateHeaderTotal()` — async fetch of `/cart.js` starts (not yet resolved)
3. `theme:cart:reload` dispatched → `loadHTML()` → `this.cart.item_count === 0` → `showEmpty()`
4. `theme:drawer:open` dispatched → `loadHTML()` → `this.stale` is now false → no-op
5. Drawer shows "empty" ✗

### Fix
Capture `updateHeaderTotal()`'s promise and chain `theme:drawer:open` inside `.then()`:

```javascript
// line 20312 — was: this.updateHeaderTotal();
const _cartUpdatePromise = this.updateHeaderTotal();

// line 20325-20327 — was: this.reloadCart() + immediate dispatchEvent
} else if (_cartDrawer) {
    _cartUpdatePromise.then(() => {
        _cartDrawer.dispatchEvent(new CustomEvent('theme:drawer:open', { bubbles: false }));
    });
}
```

### Event flow after fix
1. `/cart/add.js` — item added server-side ✓
2. `updateHeaderTotal()` → `/cart.js` fetch → `theme:cart:change` → `this.cart` updated, `this.stale = true`
3. `.then()` → `theme:drawer:open` → `loadHTML()` → `this.cart.item_count > 0` → `loadForm()` ✓

No extra network requests — `updateHeaderTotal()` was already being called; we just chain off its existing promise.

## Theme Architecture Notes (discovered this session)

### Event system
| Event | Direction | Effect |
|---|---|---|
| `theme:cart:change` | `document` (bubbles) | Updates `CartItems.this.cart` + sets `this.stale = true` |
| `theme:cart:reload` | `document` (bubbles) | Sets `this.stale = true`, calls `loadHTML()` with cached `this.cart` |
| `theme:drawer:open` | dispatched ON the drawer element (no bubble) | `CartItems` listens, calls `loadHTML()` + sets `cartOpen = true` |
| `theme:drawer:close` | dispatched ON the drawer element (no bubble) | Sets `cartOpen = false` |

### CartItems.loadHTML()
```javascript
loadHTML(plain = false) {
    if (this.stale) {
        if (this.cart && this.cart.item_count > 0) {
            this.loadForm(plain);   // renders cart items
        } else {
            this.showEmpty();       // renders empty state
        }
    }
    this.stale = false;
}
```
Critical: only renders if `this.stale === true`. Reads `this.cart.item_count` from the in-memory object, NOT from the server. Must call `updateHeaderTotal()` (or `init()`) before opening the drawer to ensure `this.cart` is current.

### Drawer open pattern
```javascript
document.querySelector('[data-drawer="drawer-cart"]')
  .dispatchEvent(new CustomEvent('theme:drawer:open', { bubbles: false }));
```

### productAddButtonForm() (Alpine)
- `x-data="productAddButtonForm()"` wrapper on any element containing a `form` and `button[name=add]`
- POSTs via axios to `/cart/add.js`
- Calls `handleSuccess()` on resolve, `handleError()` on reject
- `updateHeaderTotal()` returns a promise (axios GET to `/cart.js`)
- State machine: `INITIAL → LOADING → SUCCESS/ERROR → INITIAL` (reset after `durationAddButtonDisable` ms)

### Sticky ATC
- Vanilla JS, no Alpine
- Syncs `selling_plan` from main product form via `[name="selling_plan"]:checked` and `[name="selling-plan-group"]` listeners
- Syncs variant ID via `variant:change` custom event, MutationObserver on price element, and direct input mirror

### Recharge subscription integration
- Recharge global app embed enabled in `config/settings_data.json` (`"disabled": false`)
- Theme-native subscription block is disabled in product templates — do NOT re-enable (creates duplicate selector)
- Brand colors in Recharge config: `color_brand: #1b3731`, `contrast_color: #fff6e4`
- Selling plan ID accessed via `product.selling_plan_groups.first.selling_plans.first.id`
